// Fetch data with async/await
async function fetchUsers(endpoint) {
  try {
    const res = await fetch(endpoint);
    if (!res.ok) throw new Error(res.status);
    const data = await res.json();
    return data.users ?? [];
  } catch (err) {
    console.error("Fetch failed:", err);
    return [];
  }
}

Details & Summary — Accordion

What is semantic HTML?

Semantic HTML uses elements that convey meaning — like <nav>, <article>, <header> — rather than generic <div> containers. Improves SEO, accessibility, and maintainability.

Why does accessibility matter?

Approximately 15% of the world's population lives with some form of disability. Accessible websites ensure everyone can use your product — and often improve experience for all users.

What is the difference between id and class?

An id must be unique within a page — used for anchoring, JS targeting, ARIA. A class can be shared across multiple elements and is primarily used for CSS styling.

What is the <template> element?

The <template> element holds HTML that is not rendered until it's cloned and inserted by JavaScript. Scroll down to see a live demo where users are rendered from a template.

Can I use HTML5 elements today?

Yes! HTML5 semantic elements have near-universal browser support. Always check caniuse.com for specifics, but for semantic tags you can use them with full confidence.
